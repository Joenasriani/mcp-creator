MCP Creator

Create a real API-backed MCP server scaffold. Each generated action must call a real URL you provide. The generated server returns the real service response or a clear service error.

REAL URLS REQUIRED

01 / Define your MCP

Use a short name such as mcp-creator, store-ops-mcp, or your own app name.

Each action must call a real HTTP API. No action is generated without a real URL.

Each readable item must fetch from a real HTTP URL.

Prompts are valid MCP capabilities. They do not call an API because they are saved instruction templates.

One per line. Use names like SHOP_API_KEY. The generated code reads them from your environment.

Market-ready rule: actions and readable data must include real URLs, or they are rejected.

02 / Generated package

Add at least one real action, readable data source, or reusable instruction. Then create the MCP files.
How normal people use this
  1. Give your MCP a name.
  2. Add an action when the AI should do something through a real API, such as creating a ticket, searching products, or sending data to your backend.
  3. Add readable data when the AI should fetch information from a real URL.
  4. Add reusable instructions when you want saved prompt templates inside the MCP client.
  5. Press Create MCP files.
  6. Copy the terminal script. It writes the project files, installs dependencies, and builds the MCP server.
  7. Add your real secret keys in the terminal before running the MCP server.

For GET actions, values are added as query parameters and can also replace URL parts such as {query}. For POST, PUT, and PATCH actions, the values are sent as JSON.
